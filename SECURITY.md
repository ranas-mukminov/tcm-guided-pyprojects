# Security Policy

## Reporting a Vulnerability

If you discover a security vulnerability, please contact us via [run-as-daemon.ru](https://run-as-daemon.ru).

We take security seriously and appreciate responsible disclosure of any issues you find.

## Disclaimer

These tools are for **EDUCATIONAL PURPOSES ONLY**. Use only on systems you own or have explicit written permission to test.

**Unauthorized access to computer systems is illegal and punishable by law.**

## Responsible Disclosure

We appreciate responsible disclosure of security issues and will acknowledge contributors who follow responsible disclosure practices.

If you discover a vulnerability:
1. Contact us privately via [run-as-daemon.ru](https://run-as-daemon.ru)
2. Provide detailed information about the vulnerability
3. Allow reasonable time for us to address the issue before public disclosure
4. Do not exploit the vulnerability beyond what is necessary to demonstrate it

## Security Best Practices

When using these tools:

### Legal and Ethical
- **Always obtain written permission** before testing any system
- Use only in **isolated lab environments** for learning
- Never use on **production systems** without explicit authorization
- Follow **responsible disclosure** guidelines for any vulnerabilities found
- Respect **data privacy and confidentiality** at all times

### Technical Safety
- Run tools in **isolated virtual machines** or containers
- Use **VPN or isolated networks** for testing to avoid legal issues
- Never store **credentials or sensitive data** in code or logs
- Implement **proper error handling** to avoid exposing system information
- Use **timeouts and rate limiting** to prevent resource exhaustion

### Production Considerations
- These are **educational tools**, not production-ready solutions
- For production security testing, use **professional services**
- Implement **proper logging and monitoring** for accountability
- Follow **industry standards** (OWASP, NIST, etc.)
- Consider **legal implications** in your jurisdiction
